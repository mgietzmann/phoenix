# Question Review Guide

Use this guide to prompt the user through each section of the question template. The process is iterative — as the question evolves, revisit relevant sections rather than always starting from scratch. After any significant change to the question, note what shifted and consider which phases need re-examination.

---

## Current Question

Ask the user to state the question as it currently stands, however rough. Rough is fine — the point is to have something concrete to interrogate.

---

## Phase 1: Orient

Establish the context the question lives in. Without this, sharpening the question is working in a vacuum.

**Audience**
- Who is the primary reader? What field or background?
- What would they already accept as true going in?
- What would they be skeptical of?

**Purpose**
- If you answered this question perfectly, what argument does it support?
- What would you do with a "yes"? With a "no"?
- Is there a broader claim this question is meant to underpin?

> If the audience or purpose shifts later, return here before re-examining Sharpen.

---

## Phase 2: Sharpen

Interrogate the question itself. These tend to cascade — fixing one often surfaces the next problem — so treat this as a conversation, not a checklist.

**Specificity**
- Can any vague terms be replaced with something more concrete — a unit, a scale, a named thing?
- Is the question so specific it only applies to one case and stops generalizing?

**Falsifiability**
- What data or evidence would answer this?
- If you gathered that evidence, would the question actually be resolved?
- Is there any version of an answer that simply couldn't be obtained?

**Essentiality**
- If the answer came back as X, does your purpose follow? Or could someone accept the answer and still reject your purpose?
- Is there a more direct question that gets at the same thing?

**Causal depth**
- Are you asking what happens or why it happens?
- What picture needs painting so the reader and you are on the same page?
- What cause or mechanism needs to be established for the argument to hold?

---

## Phase 3: Situate

Zoom out. Now that the question has some shape, see how it fits with others.

**Sufficiency**
- If you had a perfect answer to this question, what would you still need to know?
- List the follow-up questions — can any be absorbed into this one with better framing?

**Related questions**
- What questions does this one depend on?
- What questions does it open up?

**Sequencing**
- What does a reader need to accept before this question makes sense?
- Are there prerequisite questions that need to come first?
- Where does this question sit in the larger argument?
